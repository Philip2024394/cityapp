# Capacitor Build — City Rider Android Driver App

The Android app is a thin Capacitor shell around the live Next.js
deployment. The native shell adds two things the browser can't:

1. **Background GPS** — keeps `current_lat/lng` fresh while the phone is
   locked or the driver is in another app. Powers movement-based busy
   detection and accurate "X km away" pricing.
2. **Foreground-service notification** — Android shows "City Rider is
   online" while the location service runs (OS requirement since
   Android 11).

The customer side stays a pure PWA — no native needed.

---

## One-time setup

### Required tools on this machine

- **Android Studio** (latest stable) — installs JDK 17 + Android SDK.
- **JAVA_HOME** env var → the JDK that Android Studio installed.
- **ANDROID_HOME** env var → typically `C:\Users\Victus\AppData\Local\Android\Sdk`.

Verify:

```powershell
java -version
$env:ANDROID_HOME
```

### Production URL — set before first build

`capacitor.config.ts` reads `CAPACITOR_SERVER_URL` if set, else falls
back to `https://cityriders.streetlocal.live`. Update that fallback (or
set the env var) to your real Vercel deploy URL before running
`cap sync`:

```powershell
$env:CAPACITOR_SERVER_URL = "https://cityriders.streetlocal.live"
npm run cap:sync
```

The URL is compiled into the native bundle. Changing it later requires
another `cap sync` + rebuild.

## Build a debug APK (install on your phone)

```powershell
# Sync web config → native project + run Gradle debug build
npm run cap:build:android
```

Output APK lands at `android/app/build/outputs/apk/debug/app-debug.apk`.
Transfer to your phone (USB or `adb install`) and install (you'll need
to allow "Install from unknown sources" in phone settings).

Faster iteration loop: open Android Studio with `npm run cap:open:android`
and use Run → connect a USB-debugging phone, hit Play.

## Build a signed release AAB (for Play Store)

### One-time: generate keystore

```powershell
keytool -genkey -v -keystore cityrider-release.keystore -alias cityrider -keyalg RSA -keysize 2048 -validity 10000
```

Store the keystore safely. **Losing it = you cannot push updates to the
same Play Store listing ever again.** Keep a backup.

### One-time: wire keystore into Gradle

Create `android/keystore.properties` (DO NOT commit):

```
storeFile=../../cityrider-release.keystore
storePassword=YOUR_KEYSTORE_PASSWORD
keyAlias=cityrider
keyPassword=YOUR_KEY_PASSWORD
```

Edit `android/app/build.gradle` and add inside `android {}`:

```gradle
def keystorePropertiesFile = rootProject.file("keystore.properties")
def keystoreProperties = new Properties()
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

signingConfigs {
    release {
        storeFile file(keystoreProperties['storeFile'])
        storePassword keystoreProperties['storePassword']
        keyAlias keystoreProperties['keyAlias']
        keyPassword keystoreProperties['keyPassword']
    }
}

buildTypes {
    release {
        signingConfig signingConfigs.release
        minifyEnabled false
    }
}
```

### Build

```powershell
npm run cap:release:android
```

Output AAB lands at `android/app/build/outputs/bundle/release/app-release.aab`.
Upload this to Play Console.

## Play Store submission checklist

1. **Google Play Console account** ($25 one-time): https://play.google.com/console
2. Create new app, package name `live.streetlocal.cityrider`.
3. **Data Safety form**:
   - Location: Precise + Coarse → Collected → Shared with no third parties → Used for App Functionality → Required for app to work.
   - Be explicit that location is used to show drivers in the marketplace.
4. **App Content** questionnaire — straightforward, declare target audience.
5. **Sensitive permissions declaration** for `ACCESS_BACKGROUND_LOCATION`:
   - Explanation: *"Drivers must be visible to customers in the local marketplace directory while the phone is in their pocket. Background location keeps them listed and ranked accurately. No trip tracking, no location history stored beyond current position."*
   - Include a 30-second screen recording of the Online toggle, lock screen, then unlock showing the driver still listed.
6. **Privacy policy URL** — must explicitly mention background location and what it's used for. Update `/legal` on the live site with this language.
7. **App icon** — 512×512 PNG (Play Store listing) + adaptive icon (already generated by Capacitor at `android/app/src/main/res/`).
8. **Feature graphic** 1024×500 PNG.
9. **Phone screenshots** — minimum 2, recommended 4–8, 1080×1920 or larger.
10. **Closed testing track first** — invite 20 drivers as testers, run for at least 14 days, then promote to production.

Expected review timeline: **3–7 days** for new apps with sensitive
permissions. Plan for one round of "please clarify your background
location usage" — respond within 48h with the same justification text
above.

## After Play Store approval

Driver onboarding flow:
1. Driver visits `cityriders.streetlocal.live` (the live web app).
2. Banner offers "Install the Android app for better location tracking".
3. Tap → opens Play Store listing → install.
4. Open app → same login as web.
5. First "Go Online" tap triggers Android's permission dialogs in order:
   - Location (While using app) → allow
   - Location (Allow all the time) → allow
   - Notifications → allow
6. Persistent "City Rider is online" notification appears.
7. Phone goes in pocket. Driver works. WhatsApp rings on incoming
   orders. Location stays fresh.

## OTA updates

App code lives at the Vercel URL, so 99% of changes ship without a Play
Store release:

- Update React/Next.js code → `git push` → Vercel deploys → driver app
  picks it up on next reload. No re-submission needed.
- The APK only needs a re-release for:
  - Native config changes (permissions, plugins, splash, icon)
  - Capacitor or plugin upgrades
  - Production URL changes

## What's wired in the Next.js code

- `capacitor.config.ts` — app ID, server.url, plugin defaults.
- `src/lib/capacitor/isNative.ts` — runtime gate (`Capacitor.isNativePlatform`).
- `src/lib/capacitor/locationBridge.ts` — `startNativeBackgroundPing` /
  `stopNativeBackgroundPing` wrapping `@capacitor-community/background-geolocation`.
- `src/lib/capacitor/initShell.ts` — one-time status bar + splash setup.
- `src/components/pwa/CapacitorBoot.tsx` — mounts initShell at root.
- `src/hooks/useDriverLocationPing.ts` — early-returns on native (bridge owns the loop).
- `src/components/rider/GoOnlineToggle.tsx` — calls native bridge on
  online/offline transitions when running native.
- `android/app/src/main/AndroidManifest.xml` — permissions block.
